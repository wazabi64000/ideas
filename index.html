Bienvenue dans le projet « Boîte à Idées »

Ce guide complet vous accompagne pas à pas dans la création d'une application fullstack moderne avec Node.js (Express ESM), MySQL et Twig. Vous apprendrez à structurer un projet MVC, créer un CRUD complet, gérer l’authentification et les rôles, permettre l’upload d’images, et construire une interface client responsive et accessible.

Résumé des étapes du projet

Étape 1 – Initiation du projet

  • Objectif : Créer l’arborescence, installer les dépendances, préparer le projet pour le développement
  • Dépendances : Express, dotenv, nodemon, mysql2, Joi, Twig, bcrypt, jsonwebtoken, multer
  • Logique / Code : Initialisation npm, structure des dossiers, configuration du serveur
  • Voir l’étape 1 complète

Étape 2 – Configuration de la base de données et modèles

  • Objectif : Créer la base MySQL, tables users et idees, et les modèles côté serveur
  • Dépendances : mysql2, dotenv
  • Logique / Code : Pool MySQL, export des modèles pour interaction avec le contrôleur
  • Voir l’étape 2 complète

Étape 3 – Validation et middlewares

  • Objectif : Sécuriser l’API en validant les données avec Joi et créer les middlewares
  • Dépendances : Joi, Express
  • Logique / Code : Schémas de validation, middleware global pour intercepter et gérer les erreurs
  • Voir l’étape 3 complète

Étape 4 – Routes et contrôleurs

  • Objectif : Créer les routes Express et relier les contrôleurs aux modèles
  • Dépendances : Express
  • Logique / Code : Définir routes GET, POST, PUT, DELETE ; utiliser les middlewares de validation
  • Voir l’étape 4 complète

Étape 5 – Authentification et gestion des rôles

  • Objectif : Sécuriser l’accès avec JWT et gérer les rôles admin/user
  • Dépendances : bcrypt, jsonwebtoken
  • Logique / Code : Hashage des mots de passe, génération et vérification du token, contrôle des rôles dans les middlewares
  • Voir l’étape 5 complète

Étape 6 – Upload d’images avec Multer

  • Objectif : Permettre aux utilisateurs d’ajouter des images aux idées
  • Dépendances : Multer
  • Logique / Code : Configuration Multer, stockage local, limite 2 Mo, filtrage pour types image
  • Voir l’étape 6 complète

Étape 7 – Intégration Twig et vues dynamiques

  • Objectif : Créer les pages HTML dynamiques côté serveur avec Twig
  • Dépendances : Twig
  • Logique / Code : Layout global, pages affichage idées, formulaire création idée
  • Voir l’étape 7 complète

Étape 8 – Affichage des idées côté client

  • Objectif : Créer l’interface utilisateur pour visualiser les idées
  • Dépendances : CSS, Twig
  • Logique / Code : Cartes d’idées responsives, affichage des informations du créateur
  • Voir l’étape 8 complète

Étape 9 – Formulaire création d’idée côté client

  • Objectif : Créer le formulaire de saisie avec upload d’image
  • Dépendances : Multer, Twig
  • Logique / Code : Formulaire HTML, validation côté client, envoi des fichiers via POST
  • Voir l’étape 9 complète

Étape 10 – Navigation et interactions JS

  • Objectif : Ajouter interactivité : menu burger, filtres, modals
  • Dépendances : JS natif ou léger framework
  • Logique / Code : Gestion événements DOM, toggles CSS/JS, navigation responsive
  • Voir l’étape 10 complète

Étape 11 – Style et accessibilité

  • Objectif : Finaliser le design, rendre le projet accessible et responsive
  • Dépendances : CSS commun, media queries
  • Logique / Code : Mobile-first, contraste, focus, boutons et formulaires accessibles
  • Voir l’étape 11 complète
